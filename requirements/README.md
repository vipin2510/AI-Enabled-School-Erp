# Adeshwar Public School — Franchise Onboarding Kit

This folder is the **starting point** for bringing a new school online in the
Pathshala ERP. Each school is a fully isolated tenant: students, fees,
attendance, library, results — none of it crosses school boundaries inside the
database.

## What's in here

```
requirements/
├── README.md            ← you are here
├── CHECKLIST.md         ← end-to-end onboarding checklist (print this)
├── IMAGE-GUIDE.md       ← how to host profile/parent images on Google Drive
├── 01-kondagaon/        ← Kondagaon (main unit)
├── 02-pharasgaon/       ← Pharasgaon branch
└── 03-chipawand/        ← Chipawand branch
```

Every school folder has the **same set of CSV templates** with headers only —
the school office fills the rows and hands the folder back. Once the data is
in, an admin (Layer 1) imports each CSV through the ERP's admin screens.

## Filling order

CSVs depend on each other (a student needs a class to exist first). Fill them
in this order:

1. `classes.csv` — every class the school runs (e.g. *Class 1, Class 2 …*).
2. `sections.csv` — sections inside each class (A, B, C …).
3. `subjects.csv` — subjects per class. Mark co-curricular ones.
4. `students.csv` — the roster. Use `class` + `section` names exactly as
   spelled in steps 1 & 2.
5. `staff.csv` — teachers, fees clerks, librarians, the principal.
6. `fee-structures.csv` + `fee-components.csv` — fee plan per class.
7. `books.csv` + `library-settings.csv` — library catalog (optional).

## Images

Profile photos, parent photos, ID-card photos — **upload to Google Drive**,
then paste the sharable link into the relevant CSV column. See
[IMAGE-GUIDE.md](IMAGE-GUIDE.md) for the exact link format and the do's and
don'ts (sharing setting, file size, naming convention).

## What you do **not** need to supply

- School name / address / board code — already coded into the ERP from the
  three franchise units.
- Login credentials — admin creates these from `staff.csv` and prints the
  phone+password handouts.
- Receipt/ID-card templates — generated by the ERP, not editable per school.
